## 网站初始页面结构规划

### 1. 首页（/）
- 导航栏
- Hero区（主标题、描述文本、CTA按钮）
- 特色课程推荐区
- 平台优势介绍区
- 页脚

### 2. 个人中心（会员中心，/dashboard）
- 用户欢迎信息
- 我的课程（学习进度）
- 用户个人设置（头像、昵称、修改密码等）

### 3. 登录与注册（/login）
- 登录/注册表单
- 第三方登录按钮（可选）
- 密码找回链接

### 4. 课程界面（/courses）
- 导航栏
- 默认展示语言与文化课程列表（课程卡片）
- 顶部两个按钮：
  - 语言与文化（默认选中，显示所有课程）
  - 风土与历史（点击后显示两个大按钮：人文历史博物馆、旅游与景观）

### 5. 单个课程详情界面（/courses/[id]）
- 课程介绍（标题、简介）
- 课程播放器
- 章节导航
- 学习进度显示（xAPI联动）

### 7. 人文历史博物馆（/humanities）
- 文化相关内容（文章、图片、视频等）
- 内容分类（可选）

### 8. 旅游与景观介绍（/travel）
- 旅游景观相关内容（文章、图片、推荐线路等）
- 内容分类（可选）

### 9. Wiki与礼仪工具页（/tools/wiki）
- 外链至其他站点（暂时外链）
- 包含藏区文化礼仪内容展示模块

### 10. 翻译与词典工具页（/tools/dictionary）
- 提供藏语翻译、藏语词典功能
- 搜索栏、查询功能模块

以上结构为初步规划，后续可根据实际需求调整和扩展。每个页面都应具备响应式设计，确保在不同设备上良好展示。

## 页面组件结构说明表（初稿）

| 页面/组件名           | 页面路径                | 功能说明                                                   | 状态管理需求            | 与后端联动/API                    |
|----------------------|-------------------------|------------------------------------------------------------|-------------------------|-----------------------------------|
| 首页 Home            | `/`                     | 展示欢迎信息、推荐课程、平台优势                           | 否                      | 无                                |
| 登录 Login           | `/login`                | 用户登录表单、密码找回                                     | ✅ 登录状态              | ✅ Supabase Auth 登录接口         |
| 用户中心 Dashboard   | `/dashboard`            | 显示用户信息、学习进度、修改密码                           | ✅ 用户Session、进度     | ✅ Supabase、xAPI（进度查询）     |
| 课程列表 Courses     | `/courses`              | 展示语言与文化课程列表、跳转人文与旅游页面                 | 否                      | ✅ Notion 数据导入、课程列表接口  |
| 单课程详情 Course    | `/courses/[id]`         | 展示课程内容、播放器、进度记录                             | ✅ 当前播放、进度记录    | ✅ Notion + xAPI 写入接口         |
| 人文历史 Humanities  | `/humanities`           | 展示文章/图片/视频等人文内容                               | 否                      | ✅ Notion 内容                    |
| 旅游与景观 Travels   | `/travel`               | 展示旅游景点介绍、推荐线路等                               | 否                      | ✅ Notion 内容                    |
| 词典 Dictionary      | `/tools/dictionary`     | 藏语词典搜索与翻译                                         | 否                      | ✅ 本地JSON词库或 API（预留）     |
| Wiki 礼仪 Wiki       | `/tools/wiki`           | 外链Wiki与礼仪内容展示                                     | 否                      | ✅ 外链站点 + Notion 礼仪条目     |

> 注：上述状态管理需求中的“✅”表示后续需统一变量管理（如登录状态、进度状态等），当前可暂缓实现，但需要预留结构。

## 页面 ↔ 功能点 ↔ API 接口 映射表（初稿）

| 页面路径              | 功能点描述                       | 调用接口/API 路径                      | 说明                                           |
|-----------------------|----------------------------------|----------------------------------------|------------------------------------------------|
| `/login`              | 用户登录验证                     | Supabase Auth                          | 前端调用 Supabase SDK，处理登录/注册逻辑       |
| `/dashboard`          | 获取当前用户信息                 | Supabase Auth                          | 从 Supabase 获取当前 session 与 profile 信息   |
| `/dashboard`          | 查询用户学习进度                 | `GET /api/v1/progress`                 | 获取 xAPI 汇总进度信息                         |
| `/courses`            | 获取课程列表                     | `GET /api/v1/courses`                  | 展示所有课程卡片（可选支持搜索/筛选）         |
| `/courses`            | 获取课程分类                     | `GET /api/v1/categories`               | 分类按钮动态获取课程类型（可后置）             |
| `/courses/[id]`       | 获取课程详情                     | `GET /api/v1/courses/:slug`            | 课程基本信息、媒体资源、结构化内容             |
| `/courses/[id]`       | 获取课程内容 JSON                | `GET /api/v1/content/:slug`            | 播放器加载课程内容结构                         |
| `/courses/[id]`       | 提交学习行为（播放记录）         | `POST /api/v1/statement`               | 播放器或交互式元素触发 xAPI 写入               |
| `/courses/[id]`       | 查询学习进度                     | `GET /api/v1/progress`                 | 获取当前课程学习记录                           |
| `/tools/dictionary`   | 查询藏语词条/翻译                | 本地 JSON / 预留 `/api/v1/dictionary` | 当前暂用本地查询，可拓展 API 支持              |
| `/tools/wiki`         | 加载藏区礼仪 Notion 数据         | Notion API Proxy                       | 当前通过 Notion 导出或代理获取                 |
| `/humanities`         | 加载人文内容（图文/视频）        | Notion API Proxy                       | 同上                                           |
| `/travel`             | 加载旅游内容（图文/推荐）        | Notion API Proxy                       | 同上                                           |
| `/checkout`（未来）   | 创建支付订单                     | `POST /api/v1/orders`                  | 支持一次性或订阅订单生成                       |
| `/checkout/result`    | 查询订单状态                     | `GET /api/v1/orders/:orderId`          | 获取支付结果展示页面内容                       |
| `/media/*`            | 加载媒体资源                     | `GET /api/v1/media/:filename`          | 可选代理至 R2 或返回签名链接                   |

> 所有 API 接口均默认前缀 `/api/v1`，如无特殊说明，采用 JSON 响应格式。
